<objective>
## Goal
Migrasi Tailwind dari CDN runtime ke PostCSS build-time, dan fix StatsCard dynamic class issue agar tidak terpurge.

## Purpose
CDN Tailwind menghasilkan console warning, tidak ada PurgeCSS (bundle besar), dan config inline di HTML tidak maintainable. PostCSS adalah setup production-standard untuk Vite. Fix StatsCard diperlukan sebelum PostCSS aktif karena `colorClass.replace('bg-', 'text-')` akan dipurge.

## Output
- `tailwind.config.ts` + `postcss.config.js` (build config)
- `index.css` (CSS entry dengan @tailwind directives)
- `index.html` dibersihkan dari CDN script, inline config, dan importmap CDN
- `components/StatsCard.tsx` menggunakan `variant` prop (static classes, purge-safe)
- App berfungsi identik secara visual seperti sebelum migrasi
</objective>

<task type="auto">
  <name>Task 3: Refactor StatsCard — colorClass string → variant enum</name>
  <files>components/StatsCard.tsx, App.tsx</files>
  <action>
    1. Update `components/StatsCard.tsx`:
       - Ganti interface props dari `colorClass: string` ke `variant: 'balance' | 'income' | 'expense'`
       - Buat static variant map (bukan dynamic string construction)
       - Update render logic

       ```tsx
       import React from 'react';
       import { LucideIcon } from 'lucide-react';
       import { formatCurrency } from '../utils/formatters';

       type StatsCardVariant = 'balance' | 'income' | 'expense';

       interface StatsCardProps {
         title: string;
         value: number;
         icon: LucideIcon;
         variant: StatsCardVariant;
       }

       const variantConfig: Record<StatsCardVariant, { container: string; icon: string }> = {
         balance: { container: 'bg-brand-500/10', icon: 'text-brand-500' },
         income:  { container: 'bg-emerald-500/10', icon: 'text-emerald-500' },
         expense: { container: 'bg-rose-500/10', icon: 'text-rose-500' },
       };

       export const StatsCard: React.FC<StatsCardProps> = ({ title, value, icon: Icon, variant }) => {
         const { container, icon } = variantConfig[variant];
         return (
           <div className="bg-white dark:bg-gray-800 rounded-xl shadow-sm p-6 border border-gray-100 dark:border-gray-700">
             <div className="flex items-center justify-between">
               <div>
                 <p className="text-sm font-medium text-gray-500 dark:text-gray-400">{title}</p>
                 <h3 className="text-2xl font-bold mt-2 text-gray-900 dark:text-white">
                   {formatCurrency(value)}
                 </h3>
               </div>
               <div className={`p-3 rounded-lg ${container}`}>
                 <Icon className={`w-6 h-6 ${icon}`} />
               </div>
             </div>
           </div>
         );
       };
       ```

    2. Update `App.tsx` — ganti prop pada semua penggunaan StatsCard:
       - Cari: `<StatsCard ... colorClass="bg-brand-500 text-brand-500" />`
       - Ganti: `<StatsCard ... variant="balance" />`
       
       Saat ini hanya ada 1 usage di App.tsx (line ~355). Ganti colorClass → variant="balance".
  </action>
  <verify>
    `StatsCard.tsx` tidak mengandung string `colorClass` atau `.replace('bg-', 'text-')`.
    `App.tsx` menggunakan `variant="balance"` bukan `colorClass`.
    TypeScript tidak menampilkan error: `npx tsc --noEmit` (atau cek IDE diagnostics).
  </verify>
  <done>AC-3 satisfied: StatsCard menggunakan static class strings yang aman dari Tailwind purge.</done>
</task>
