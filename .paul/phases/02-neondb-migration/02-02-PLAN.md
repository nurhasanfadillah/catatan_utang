<objective>
## Goal
Buat 6 Vercel serverless functions di /api/ yang meng-query NeonDB via Drizzle — sehingga DATABASE_URL tidak pernah menyentuh browser.

## Purpose
Saat ini services/api.ts dan services/auth.ts memanggil Supabase langsung dari browser. Setelah plan ini selesai, ada API layer yang berdiri di server: DATABASE_URL hanya ada di Vercel Environment Variables, dan browser cukup memanggil /api/*. Plan 02-03 akan merewrite frontend untuk memanggil route-route ini.

## Output
- `api/transactions/index.ts` — GET (list + filter + pagination) + POST (create)
- `api/transactions/[id].ts` — PUT (update) + DELETE
- `api/transactions/summary.ts` — GET total income/expense/balance
- `api/transactions/net-change.ts` — GET net change untuk N rows (dipakai perhitungan pagination balance)
- `api/transactions/export.ts` — GET semua transaksi untuk PDF export
- `api/auth/login.ts` — POST login via NeonDB (pengganti Supabase query di auth.ts)
- `vercel.json` diupdate agar /api/* tidak kena SPA rewrite
- `package.json` ditambah `@vercel/node` untuk types
</objective>

<task type="auto">
  <name>Task 1: Setup + semua transaction API routes</name>
  <files>package.json, vercel.json, api/transactions/index.ts, api/transactions/[id].ts, api/transactions/summary.ts, api/transactions/net-change.ts, api/transactions/export.ts</files>
  <action>
    Step 1 — Update package.json: tambah "@vercel/node": "^5.0.0" ke devDependencies, lalu jalankan `npm install`.

    Step 2 — Update vercel.json: ganti rewrite agar /api/* tidak kena SPA redirect:
    ```json
    {
      "rewrites": [
        { "source": "/((?!api/).*)", "destination": "/index.html" }
      ]
    }
    ```

    Step 3 — Buat api/transactions/index.ts:
    Handler untuk GET (list) dan POST (create). Import `db` dari `../../db/index`, import tabel `transactions` dari `../../db/schema`, import operator `{ eq, gte, lte, desc, count, and }` dari `drizzle-orm`.

    GET: baca query params `page` (default '1'), `pageSize` (default '10'), `filterType` (default 'ALL'), `startDate`, `endDate`. Bangun kondisi where dengan Drizzle operators. Jalankan dua query paralel: select+limit+offset (untuk data) dan select count (untuk total). Return `{ data, count }`. Map setiap row: tambahkan `createdAt: item.created_at`, convert `amount: Number(item.amount)`. Sort: desc(date), desc(created_at).

    POST: parse `req.body` sebagai `{ date, description, type, amount }`. Insert ke transactions dengan `created_at: Date.now()`. Return 201 + row yang baru dibuat (dengan `createdAt` mapping dan `amount` as Number).

    Method selain GET/POST → 405.

    Step 4 — Buat api/transactions/[id].ts:
    PUT: ambil `req.query.id`, parse body, update row dengan `eq(transactions.id, id)`. Jangan allow update `id` dan `created_at`. Return 200.
    DELETE: ambil `req.query.id`, delete dari transactions WHERE id = id. Return 204.
    Method lain → 405.

    Step 5 — Buat api/transactions/summary.ts:
    GET: select semua `{ type, amount }` dari transactions. Hitung `income` (type = 'TAGIHAN') dan `expense` (type = 'KASBON') dengan `Number(t.amount)`. Return `{ income, expense, balance: income - expense }`.

    Step 6 — Buat api/transactions/net-change.ts:
    GET: baca query params `limit`, `filterType`, `startDate`, `endDate` (sama persis seperti getTransactions). Jika limit <= 0, return { netChange: 0 }. Bangun kondisi where yang sama. Select `{ type, amount }` dari transactions, sort sama, range 0..limit-1. Hitung netChange: TAGIHAN tambah, KASBON kurangi. Return `{ netChange }`.

    Step 7 — Buat api/transactions/export.ts:
    GET: baca query params `startDate`, `endDate` (optional). Select semua transactions dengan optional date filter. Sort desc date, desc created_at. Map rows dengan `createdAt` dan `amount` as Number. Return array Transaction[].

    PENTING untuk semua routes:
    - Wrap seluruh handler dalam try/catch, kembalikan 500 + { error: err.message } jika error
    - Jangan import dari types.ts frontend — gunakan string literals ('TAGIHAN', 'KASBON')
    - db/index.ts adalah import server-only; sudah aman karena routes ini berjalan di server
    - Semua responses harus `res.setHeader('Content-Type', 'application/json')` atau gunakan `res.json()`
  </action>
  <verify>
    Jalankan `npm run build` — harus sukses tanpa error baru.
    Jalankan `npx tsc --noEmit` — tidak ada type error.
    Pastikan semua 5 file di api/transactions/ dan vercel.json terbuat/terupdate.
  </verify>
  <done>AC-1 partial (files ada, build clean) + AC-4 satisfied (vercel.json updated)</done>
</task>

<task type="auto">
  <name>Task 2: Auth API route</name>
  <files>api/auth/login.ts</files>
  <action>
    Buat api/auth/login.ts:
    - Hanya accept POST; method lain → 405
    - Parse `req.body` sebagai `{ username, password }`
    - Jika username atau password kosong → 400 + { error: 'Username dan password wajib diisi' }
    - Import `db` dari `../../db/index`, `appUsers` dari `../../db/schema`, `{ eq, and }` dari `drizzle-orm`
    - Query: `db.select().from(appUsers).where(and(eq(appUsers.username, username), eq(appUsers.password, password)))`
    - Jika tidak ada hasil (array kosong) → 401 + { error: 'Username atau password salah' }
    - Jika ada hasil → 200 + { username: user.username, name: user.name, role: user.role }
    - Wrap dalam try/catch → 500 + { error: err.message }

    CATATAN: Backdoor login (admin/root) tetap di services/auth.ts frontend — tidak dipindah ke sini.
    Route ini hanya handle query DB. Tidak ada session/JWT — frontend tetap pakai localStorage (tidak berubah di plan ini).
  </action>
  <verify>
    Jalankan `npx tsc --noEmit` — tidak ada type error di api/auth/login.ts.
    Pastikan file api/auth/login.ts terbuat dengan benar.
  </verify>
  <done>AC-3 satisfied (route ada, query appUsers via Drizzle, return format benar)</done>
</task>
