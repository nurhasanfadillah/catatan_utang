<objective>
## Goal
Rewrite `services/api.ts` dan `services/auth.ts` agar memanggil Vercel API routes (`/api/*`) sebagai pengganti Supabase SDK — lalu hapus semua Supabase dari codebase.

## Purpose
Plan 02-02 membuat API layer di server. Plan ini menyelesaikan migrasi dengan memutus koneksi browser ke Supabase: tidak ada lagi `supabase.co` request dari frontend, `DATABASE_URL` sepenuhnya server-only, dan dependency `@supabase/supabase-js` dihapus dari bundle.

## Output
- `services/api.ts` — semua 7 functions diganti dengan `fetch()` ke `/api/*`
- `services/auth.ts` — Supabase DB query diganti dengan `POST /api/auth/login`; backdoor tetap
- `services/supabaseClient.ts` — dihapus
- `sw.js` — hapus Supabase dari cache list, ganti network-only rule ke `/api/*`
- `package.json` — hapus `@supabase/supabase-js` dependency
</objective>

<task type="auto">
  <name>Task 1: Rewrite services/api.ts — Supabase ke fetch()</name>
  <files>services/api.ts</files>
  <action>
    Hapus `import { supabase } from './supabaseClient'` dan `import { Transaction, TransactionType } from '../types'` (TransactionType masih diperlukan, tapi supabase tidak).

    Pertahankan import TransactionType dari '../types'.

    Ganti implementasi setiap function dengan fetch() ke Vercel API routes. Gunakan pola berikut:

    **Helper internal (opsional tapi direkomendasikan):**
    ```typescript
    const BASE = '/api';
    async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
      const res = await fetch(`${BASE}${path}`, init);
      if (!res.ok) throw new Error(`API error ${res.status}`);
      return res.json();
    }
    ```

    **getTransactions(page, pageSize, filterType, startDate, endDate):**
    - `GET /api/transactions?page=N&pageSize=N&filterType=X&startDate=Y&endDate=Z`
    - Build URLSearchParams dari params
    - Return `{ data, count }` — mapping date: normalize ISO string ke YYYY-MM-DD karena NeonDB return format `2026-06-15T00:00:00.000Z`:
      ```typescript
      data: raw.data.map((item: any) => ({
        ...item,
        date: typeof item.date === 'string' ? item.date.split('T')[0] : item.date,
        amount: Number(item.amount),
        createdAt: item.createdAt ?? item.created_at
      }))
      ```

    **getBalanceSummary():**
    - `GET /api/transactions/summary`
    - Return `{ income, expense, balance }` langsung dari response

    **getNetChangeSum(limit, filterType, startDate, endDate):**
    - `GET /api/transactions/net-change?limit=N&filterType=X&startDate=Y&endDate=Z`
    - Return `response.netChange` (number)

    **getAllTransactionsForExport(startDate?, endDate?):**
    - `GET /api/transactions/export?startDate=Y&endDate=Z` (skip param jika undefined)
    - Return array langsung, sama mapping date normalization seperti getTransactions

    **createTransaction(transaction):**
    - `POST /api/transactions` dengan body JSON `{ date, description, type, amount }`
    - Return response dengan date normalization

    **updateTransaction(id, transaction):**
    - `PUT /api/transactions/${id}` dengan body JSON `{ date, description, type, amount }`
    - Exclude `createdAt` dan `id` dari body (sama seperti sebelumnya)
    - Return void

    **deleteTransaction(id):**
    - `DELETE /api/transactions/${id}`
    - Return void

    PENTING:
    - Semua fetch harus `Content-Type: application/json` untuk POST/PUT
    - Error handling: jika `!res.ok`, throw Error dengan status code
    - Jangan import dari supabaseClient — file itu akan dihapus di Task 3
  </action>
