<objective>
## Goal
Migrate deployment dari Netlify ke Vercel dengan custom domain `kaspro.redone.my.id`.

## Purpose
Menggantikan Netlify sepenuhnya sebagai platform deployment. Vercel dipilih karena CLI/MCP sudah terkonfigurasi dan mendukung Vite SPA out of the box.

## Output
- `vercel.json` — SPA routing config untuk Vercel
- `_redirects` — dihapus (Netlify-specific, tidak dibutuhkan di Vercel)
- App live di `kaspro.redone.my.id` dengan HTTPS
</objective>

## AC-1: SPA Routing Berfungsi di Vercel
```gherkin
Given app sudah di-deploy ke Vercel

Then halaman tetap tampil (bukan 404)
```

## AC-2: App Live di Custom Domain
```gherkin
Given CNAME record `kaspro.redone.my.id` → Vercel sudah aktif
When user membuka https://kaspro.redone.my.id di browser
Then app Keuangan Produksi tampil dengan HTTPS valid (tidak ada warning SSL)
```

## AC-3: Netlify Tidak Lagi Aktif
```gherkin
Given migrasi selesai
When deployment dilakukan
Then hanya Vercel yang menerima deploy (tidak ada config Netlify aktif)
```

<tasks>

<task type="auto">
  <name>Task 1: Buat vercel.json dan hapus _redirects</name>
  <files>vercel.json, _redirects</files>
  <action>
    1. Buat file `vercel.json` di root project dengan konten:
    ```json
    {
      "rewrites": [
        { "source": "/(.*)", "destination": "/index.html" }
      ]
    }
    ```
    Ini menggantikan fungsi `_redirects` milik Netlify agar SPA routing (refresh halaman selain root) tidak menghasilkan 404.

    2. Hapus file `_redirects` — file ini Netlify-specific dan tidak dibutuhkan di Vercel.

    Avoid: Jangan tambahkan `buildCommand` atau `outputDirectory` di vercel.json — Vercel auto-detect Vite.
  </action>
  <verify>
    - `vercel.json` ada di root dan berisi rewrites ke `/index.html`
    - `_redirects` tidak ada lagi di root directory
  </verify>
  <done>AC-1 satisfied: SPA routing config siap untuk Vercel</done>
</task>

<task type="auto">
  <name>Task 2: Deploy ke Vercel dan tambah custom domain</name>
  <files>— (infrastruktur, tidak ada file kode yang diubah)</files>
  <action>
    1. Deploy project ke Vercel production via CLI:
    ```bash
    vercel --prod
    ```
    Ikuti prompt jika ini deploy pertama (link ke project baru atau existing).
    Catat URL deployment default (format: `*.vercel.app`).

    2. Setelah deploy sukses, tambah custom domain via CLI:
    ```bash
    vercel domains add kaspro.redone.my.id
    ```

    3. Vercel akan menampilkan instruksi DNS. Catat nilai yang diminta:
    - Biasanya: CNAME `kaspro` → `cname.vercel-dns.com`
    - Atau: A record jika apex domain

    Tampilkan DNS instructions kepada user untuk Task 3.
  </action>
  <verify>
    - `vercel --prod` selesai tanpa error
    - App dapat diakses di URL `*.vercel.app` default
    - Output `vercel domains add` menampilkan DNS instructions
  </verify>
  <done>AC-3 satisfied: App terdeploy di Vercel dan domain sudah di-register</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <what-to-do>
    Tambah CNAME record di Cloudflare DNS untuk domain `kaspro.redone.my.id`.

    Berdasarkan instruksi dari Task 2 (output `vercel domains add`), tambahkan record berikut di Cloudflare dashboard (https://dash.cloudflare.com):

    1. Buka DNS settings untuk domain `redone.my.id`
    2. Tambah record baru:
       - **Type:** CNAME
       - **Name:** `kaspro`
       - **Target:** `cname.vercel-dns.com` (atau nilai dari output Vercel)
       - **Proxy status:** DNS only (gray cloud) — jangan aktifkan Cloudflare proxy dulu agar SSL Vercel dapat provisioning

    3. Save record

    Propagasi DNS membutuhkan waktu 1–10 menit.
  </what-to-do>
  <resume-signal>Ketik "done" setelah CNAME record ditambahkan di Cloudflare</resume-signal>
</task>
